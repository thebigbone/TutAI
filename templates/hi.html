Ansible Tutorial

Welcome to the Ansible tutorial! Ansible is an open-source automation tool that allows you to automate tasks on multiple servers. In this tutorial, we will cover some of the basic commands used in Ansible.

Getting Started

  • Install Ansible: You can install Ansible on your local machine using pip or yum. For more information, visit the Ansible website.
  • Create an inventory file: An inventory file is used to define the hosts and variables used in your Ansible playbook. Create a new file called "inventory" in the same directory as your playbook. Here's an example:

    ```yaml [webservers] server1 ansible_host=192.168.0.100 server2 ansible_host=192.168.0.101 [dbservers] db1 ansible_host=192.168.0.102 db2 ansible_host=192.168.0.103 ```

Basic Commands

  • ansible-playbook: This command is used to run an Ansible playbook. Here's an example:

    ```yaml ansible-playbook -i inventory playbook.yml ``` This will run the "playbook.yml" playbook on the servers defined in the "inventory" file.
  • ansible-inventory: This command is used to list the hosts and variables defined in an inventory file. Here's an example:

    ```yaml ansible-inventory -i inventory ``` This will display a list of all the hosts and variables defined in the "inventory" file.
  • ansible-var: This command is used to view the value of a variable defined in an inventory file. Here's an example:

    ```yaml ansible-var -i inventory webservers.db1_user ``` This will display the value of the "db1\_user" variable for the "webservers.db1" host.
